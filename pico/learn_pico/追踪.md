# pico 运行追踪文件说明

每次 pico 执行一次用户请求（一次 ask()），会在 `.pico/runs/{run_id}/` 目录下生成三个文件。
它们共同记录了"这次运行做了什么、怎么做的、最后结论是什么"。

---

## 三个文件的定位

| 文件 | 视角 | 更新时机 |
|---|---|---|
| `task_state.json` | 当前进度的极简快照，只有结论性字段 | 每次工具执行后实时覆盖 |
| `trace.jsonl` | 完整过程，每一步追加一行，含工具参数/结果/耗时 | 每个事件实时追加 |
| `report.json` | 最终摘要，含 prompt 指标、记忆提升、脱敏 env 等 | 运行结束时一次性写入 |

---

## task_state.json — 进度快照

回答："这次请求现在进行到哪了？"
运行中每执行一次工具就覆盖写一次，所以可以实时观察，也可以事后复盘。

### 字段说明

```json
{
  "run_id":       "run_20260529-104314-edfea5",   // 本次运行唯一 ID（时间戳+随机hex）
  "task_id":      "task_20260529-104314-211ca2",  // 本次任务 ID
  "user_request": "查看一下当前项目做了什么",         // 用户原始输入

  "status":       "completed",                    // 运行状态（见下方状态机）
  "stop_reason":  "final_answer_returned",        // 停止原因（见下方枚举）

  "tool_steps":   2,                              // 真正执行了几次工具
  "attempts":     3,                              // 模型被调用了几轮（含重试）
  "last_tool":    "read_file",                    // 最后执行的工具名

  "final_answer": "...",                          // 模型给出的最终回答文本

  "checkpoint_id":  "ckpt_a3e1e32c",             // 最后一个 checkpoint 的 ID
  "resume_status":  "no-checkpoint"              // 恢复状态（这次是全新运行）
}
```

> 注意：`attempts: 3` vs `tool_steps: 2` 说明模型被调了 3 轮，其中 2 轮执行了工具，第 3 轮直接给出最终答案。

### status 状态机

```
running  →  completed   正常结束，给了最终答案
running  →  stopped     触发上限或被拒绝
running  →  failed      模型 API 报错
```

### stop_reason 枚举

| 值 | 含义 |
|---|---|
| `final_answer_returned` | 正常完成 |
| `step_limit_reached` | 工具调用次数达到上限 |
| `retry_limit_reached` | 输出格式错误重试次数耗尽 |
| `model_error` | 模型 API 调用失败 |
| `tool_timeout` | 工具执行超时 |
| `approval_denied` | 用户拒绝了危险操作 |
| `delegate_failed` | 子 agent 失败 |
| `persistence_error` | 持久化写入失败 |
| `resume_load_error` | 恢复 checkpoint 失败 |

## trace.jsonl — 逐事件时间线

回答："这次运行一步一步具体做了什么？"
每行是一个 JSON 事件，按时间顺序追加，不会覆盖。
适合调试，能还原 agent 的完整决策过程。

### 事件类型

| event | 触发时机 | 关键字段 |
|---|---|---|
| `run_started` | 运行开始 | `task_id`, `user_request` |
| `prompt_built` | 每轮向模型发请求前，prompt 构建完成 | `prompt_metadata`（字符数、缓存key、各section大小） |
| `model_requested` | 开始调用模型 API | `attempts`, `tool_steps`, `prompt_cache_key` |
| `model_parsed` | 模型输出被解析 | `kind`（tool/final/retry）, `duration_ms` |
| `tool_executed` | 一次工具调用完成 | `name`, `args`, `result`, `duration_ms`, `tool_status` |
| `checkpoint_created` | checkpoint 写入完成 | `checkpoint_id`, `trigger` |
| `run_finished` | 运行正常结束 | `final_answer`, `status`, `run_duration_ms` |

### tool_executed 事件详解

这是最重要的事件，记录每一次工具调用的完整信息：

```json
{
  "event":        "tool_executed",
  "name":         "list_files",           // 工具名称
  "args":         {"path": "."},          // 传入的参数
  "result":       "[D] pico\n[F] README.md...", // 工具返回结果（截断至500字符）
  "duration_ms":  0,                      // 执行耗时（毫秒）
  "tool_status":  "ok",                   // ok / partial_success / error / rejected
  "read_only":    true,                   // 是否只读操作
  "risk_level":   "low",                  // 风险等级
  "affected_paths": [],                   // 被修改的文件路径
  "workspace_changed": false,             // 执行后工作区是否变化
  "workspace_fingerprint": "e805..."      // 工作区状态指纹
}
```

### 一次完整请求的 trace 事件序列

```
run_started
prompt_built        ← 第1轮 prompt 构建
model_requested     ← 第1轮调用模型
model_parsed        ← 解析结果: kind=tool
tool_executed       ← 执行 list_files
checkpoint_created  ← 工具执行后创建 checkpoint
prompt_built        ← 第2轮 prompt 构建（带上工具结果）
model_requested     ← 第2轮调用模型
model_parsed        ← 解析结果: kind=tool
tool_executed       ← 执行 read_file
checkpoint_created  ← 工具执行后创建 checkpoint
prompt_built        ← 第3轮 prompt 构建
model_requested     ← 第3轮调用模型
model_parsed        ← 解析结果: kind=final
checkpoint_created  ← 运行结束前创建最终 checkpoint
run_finished        ← 运行结束
```

## report.json — 最终运行摘要

回答："这次运行的结论和关键指标是什么？"
运行结束后一次性写入，不会实时更新。
适合批量分析和人工审计。

### 顶层字段

```json
{
  "run_id":        "run_20260528-145730-9c613e",
  "task_id":       "task_20260528-145730-00784f",
  "status":        "stopped",
  "stop_reason":   "step_limit_reached",
  "final_answer":  "Stopped after reaching the step limit...",
  "tool_steps":    6,                        // 本次共执行了几次工具
  "attempts":      6,                        // 模型被调用了几轮
  "checkpoint_id": "ckpt_81b34f0f",
  "resume_status": "full-valid",

  "task_state":    { ... },                  // task_state.json 内容的副本

  "durable_promotions": [],                  // 本次运行提升到长期记忆的条目
  "durable_rejections": [],                  // 被拒绝的长期记忆提升
  "durable_superseded": [],                  // 被覆盖的长期记忆条目

  "redacted_env":  { ... },                  // 检测到的敏感环境变量摘要（脱敏）
  "prompt_metadata": { ... }                 // 最后一轮 prompt 的详细指标（见下）
}
```

### prompt_metadata 字段说明

记录最后一轮 prompt 构建的详细信息，用于分析上下文压缩效果：

```json
{
  "prompt_chars":        6018,    // 实际发给模型的总字符数
  "prompt_budget_chars": 12000,   // 总字符预算上限
  "prompt_over_budget":  false,   // 是否超出预算

  "prefix_chars":  5062,          // system prompt 部分字符数
  "history_chars": 3883,          // 对话历史部分字符数
  "memory_chars":  103,           // 工作记忆部分字符数

  "prefix_changed":     false,    // 这轮 prefix 相比上轮是否变化
  "prefix_hash":        "2bed...", // prefix 内容的 SHA256，用于缓存复用
  "prompt_cache_key":   "2bed...", // 模型端 prompt 缓存 key

  "tool_count":    7,             // 注册的工具数量
  "tool_signature": "4426...",    // 工具集签名（工具变化时会改变）
  "workspace_fingerprint": "4229...", // 工作区状态指纹

  "section_budgets": {            // 各 section 的字符预算
    "prefix":           3600,
    "memory":           1600,
    "relevant_memory":  1200,
    "history":          5200,
    "current_request":  null
  },
  "budget_reductions": [],        // 超预算时裁剪了哪些 section

  "history": {
    "raw_chars":              11293,  // history 原始字符数
    "rendered_chars":          2253,  // 压缩后实际使用字符数
    "summarized_tool_count":      7,  // 被摘要化的工具调用数
    "collapsed_duplicate_reads":  0   // 被折叠的重复读取次数
  }
}
```

## 快速查看命令（PowerShell）

```powershell
# 找到最新的 run 目录
$latest = (Get-ChildItem .pico\runs | Sort-Object LastWriteTime -Descending | Select-Object -First 1).FullName

# 查看进度快照
Get-Content "$latest\task_state.json" | ConvertFrom-Json

# 只看工具调用事件
Get-Content "$latest\trace.jsonl" |
  ForEach-Object { $_ | ConvertFrom-Json } |
  Where-Object { $_.event -eq "tool_executed" } |
  Select-Object event, name, duration_ms, tool_status

# 查看最终报告摘要
Get-Content "$latest\report.json" | ConvertFrom-Json |
  Select-Object status, stop_reason, tool_steps, attempts
```
